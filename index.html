TCGA-KICH Cancer Classification Pipeline

End-to-end ML pipeline classifying kidney chromophobe carcinoma (KICH) -tumor vs. normal -from TCGA RNA-seq gene expression data. 91 patients, 43,806 genes (p ≫ n). Rigorous, leak-free sklearn pipeline with a full feature-selection and cross-validation audit.

ProblemCancer transcriptomic data is ultra-high-dimensional and leakage-prone -naive ML pipelines overfit catastrophically and report misleading accuracy.
RoleSole engineer -EDA, feature engineering, modeling, validation.
PipelineLog₂(x+1) normalization → variance threshold → SelectKBest (ANOVA F-test) → RFE. Stratified split, leak-free sklearn Pipeline, GridSearchCV inside CV, learning-curve diagnostics. PCA + t-SNE for latent structure; Mahalanobis distance for outlier detection.
OutcomeNear-perfect classification across Logistic Regression, Random Forest, and MLP. Only 5 genes sufficient for ceiling-level performance -identified biologically discriminative gene signatures and quantified the interpretability vs. complexity tradeoff.

11SOFTWARE · DATA STRUCTURES
↗
🗂️

University Course Registration System

Java course-registration platform showing how a Hash Table, Binary Search Tree, and Linked Queue collaborate to manage courses, enrollment, and waiting lists - with full object serialization so state survives between runs.

ProblemModel real university registration - O(1) course lookup by CRN, ordered student rosters per course, fair FIFO waiting lists when sections fill, and durable state across sessions - without leaning on a library or a database.
RoleSole engineer -data structure design, deletion semantics, persistence, and the CLI driver.
ArchitectureOpen-addressed Hash Table (linear probing, hash = crn % size) with a defunct sentinel for lazy/tombstone deletion stores every Course. Each course owns a generic Binary Search Tree of enrolled students keyed by Student ID - supporting insert, search, full delete (including the two-child in-order-successor case), and pre/in/post-order traversal. Each course also owns a FIFO queue backed by a singly linked list with head/tail pointers for O(1) enqueue/dequeue, used as the waiting list. Dropping a student auto-promotes the head of the queue into the freed seat; raising capacity drains the queue until full.
PersistenceEvery entity (TreeHashTable, Course, Student, BST nodes, queue nodes) implements java.io.Serializable. The full hash table → courses → BSTs + queues graph round-trips through details.dat via ObjectInputStream / ObjectOutputStream on launch and exit.
OutcomeInteractive console with 8 operations: add course, enroll student (auto-queue when full), drop (auto-promote from waitlist), raise capacity (drain waitlist), in-order roster display, cross-course student lookup, per-course Enrolled vs. WaitingList status, and a serialized save-on-exit. Defensive input validation throughout.
